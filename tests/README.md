# Tests

Top-level test coordination for the ourRAG MVP.

Use this directory for cross-service and end-to-end tests. Backend unit and integration tests should live in `backend/tests/`; frontend tests should live in `frontend/tests/`.

Planned categories:

- `backend/tests/` for API, domain, persistence, and worker tests.
- `frontend/tests/` for React component and UI workflow tests.
- `tests/e2e/` for full local-stack scenarios once the API and UI exist.
